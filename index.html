Two interfaces — a PyQt5 GUI and a shell script — for bypassing FRP on Samsung Galaxy devices via ADB and AT serial commands.

Getting started

PyQt5 GUI

  • 1 Install Python 3.10+
  • 2 Install dependencies:
    pip install -r requirements.txt
  • 3 Connect your Samsung device over USB
  • 4 Launch the tool:
    python main.py

unlock.sh

  • 1 Clone the repo and enter the directory
  • 2 Make the script executable:
    chmod +x unlock.sh
  • 3 Connect device, then run:
    ./unlock.sh
  • 4 Follow on-screen prompts to complete factory reset 
# Clone and run
git clone https://github.com/sudo-self/samsung-frp.git
cd samsung-frp
chmod +x unlock.sh
./unlock.sh
ADB command sequence

What the bypass does

Writes directly to Android's settings database to mark the device as fully provisioned, bypassing the setup wizard. Then opens Settings for a factory reset to clear FRP entirely.

Setup Wizard
settings put global setup_wizard_has_run 1
User Setup
settings put secure user_setup_complete 1
Provisioned
content insert … DEVICE_PROVISIONED 1
Setup Complete
content insert … user_setup_complete 1
Sideloading
content insert … INSTALL_NON_MARKET_APPS 1
Home Screen
am start -c android.intent.category.HOME
Wait 5s →
am start -n com.android.settings/com.android.settings.Settings
Wait 5s → Reboot
adb shell reboot
